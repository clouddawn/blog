# 菲茨定律

### 1. 核心定义

**菲茨定律（Fitts's Law）** 预测：**用手指或鼠标点击一个目标所需的时间，只取决于两个变量——目标离你有多远（距离），以及目标本身有多大（尺寸）。**

**距离越短、目标越大，点击越快，失误越少；反之则越慢、失误越多。**

------

### 2. 数学公式

如果你去查论文，公式是：**T = a + b · log₂(D/W + 1)**

别被吓到，拆开看就三个东西：

- **T**：点击花费的时间（我们想让它变小）。
- **D（Distance）**：起点到目标中心的**距离**。
- **W（Width）**：目标在移动方向上的**宽度**（尺寸）。
- **a 和 b**：是常数（人的反应速度，不用管）。

**核心精髓就是 log₂(D/W + 1)**。这个对数关系告诉我们：**把目标放大一倍，或者把距离缩短一半，点击效率会呈对数级别提升**，效果非常显著。

------

### 3. 这条定律推导出的“交互三大铁律”

基于这个公式，在产品设计中会直接得出三条不容置疑的结论：

- **铁律一：能近就近（减少 D）**
  操作按钮要紧挨着用户当前的操作点。比如，你在 iPhone 上打字时，候选词和键盘紧紧贴在一起，不用移动视线和手指去远处点击；右键菜单（Context Menu）永远出现在鼠标旁边，而不是屏幕顶部，这就是“就近原则”的极致体现。
- **铁律二：能大就大（增大 W）**
  核心操作按钮（如“提交订单”、“发送”）一定要做得足够大。但要注意，这里说的“大”是指**点击热区（Hit Area）**，不一定是视觉大小。比如很多 App 的列表条目，视觉上只有一行字，但实际点击热区横贯整个屏幕宽度，这就是在利用规则降低点击难度。
- **铁律三：边缘“作弊”**（这是最精妙的一点）
  电脑屏幕的**四角**和**四边**是“无限大”的按钮！因为鼠标指针被屏幕边缘挡住无法移出去，所以不管光标从多远的地方滑向屏幕左上角的“关闭”按钮，实际上永远不会移过头。在菲茨定律计算中，屏幕四角的 W（宽度）被视为**无穷大**，所以点击成本极低。这也是为什么 Mac 把菜单栏固定在顶部，Windows 把“开始”菜单放在左下角——它们利用了物理屏幕的边框来降低操作难度。

------

### 4. 经典正反案例对比

- **正面案例（Win10 的“开始”菜单 vs Mac 的 Dock 栏）**：
  Win10 开始菜单在左下角（贴边），鼠标甩过去就能精准点到，不需要瞄准；Mac 的 Dock 栏在底部中央（不贴边），你必须从中间位置精准移动到那个小图标上，操作成本远高于 Windows 的开始菜单（这也是很多人觉得 Mac 窗口不好缩放的原因之一）。
- **反面案例（折叠的“更多”菜单）**：
  很多老版 App 把高频操作（比如“删除”）藏在右上角的“...”里面。用户要从当前操作区域（比如屏幕中间）把手指/鼠标挪到右上角（D 很大），并且“...”图标极小（W 很小）。根据菲茨定律，这样设计效率极低。现在的设计趋势是把高频操作放在**底部弹出式菜单（Action Sheet）**，手指不用移动太远，且选项呈长条形（W 变大）。

------

### 5. 最重要的误区和禁忌

**误区：既然越大越好，那我干脆把按钮铺满全屏？**
错。虽然菲茨定律支持放大，但**界面空间是稀缺资源**，且**视觉干扰**（认知负荷）会随之增加。如果所有按钮都巨大，用户反而找不到重点。

**禁忌：当“安全”遇上“效率”时，菲茨定律要让步！**
这就是我们之前提到的**防错原则**。

- 比如 **“删除账号”** 或 **“付款”** 按钮。如果严格按照菲茨定律，它应该又大又近。但真实设计中，我们反而会**刻意把它做小、放远**，甚至增加二次确认弹窗。
- 这说明：**菲茨定律解决的是“好不好点”的问题，但“能不能点”由业务和安全策略决定。** 如果方案B（小且远）是为了防止用户手滑造成巨大损失，那么即便它违反菲茨定律，它也是更优的交互逻辑。

------

### 6. 实战自检清单

下次拿到两个交互方案，把手指或鼠标放在“当前所在位置”，问自己三个问题：

1. 我的手指/鼠标到达目标按钮，**最短路径**是多少？（有没有更近的摆放位置？）
2. 目标按钮的**点击热区**，有没有被文字/图片边缘浪费掉？（是不是只点了文字，其实整行都可以点？）
3. 如果我把这个按钮做成**贴底**或**贴边**，能不能利用屏幕边框把它变成“无限大”？
