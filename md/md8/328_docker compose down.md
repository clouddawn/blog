# docker compose down

`docker compose down` 就是 Docker Compose 里用来清楚环境的命令，它会把 `docker compose up` 创建出来的容器、网络等资源停掉并删除，让系统回到启动前的状态。

## `down` 与 `stop` 的区别

| 命令                  | 说明                                                         | 适用场景                                                     |
| --------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| `docker compose stop` | 仅停止容器，但不删除。所有资源、数据卷、网络都保留。         | 只是想暂停服务，之后可以快速用 `docker compose start` 恢复。 |
| `docker compose down` | 停止并删除容器及网络，环境被彻底重置。默认会保留数据卷（即关键数据还在），下次 `up` 会生成全新的容器。 | 想要一个干净的测试环境，或清除整个项目。                     |

## 基础语法与行为

```bash
docker compose down [OPTIONS] [SERVICES]
```

在不加任何参数执行时，`down` 命令会帮你完成三件清理工作：

* 停止并删除容器：所有在 `compose.yaml` 文件中定义的服务容器都会被删除。
* 删除网络：由 Compose 项目创建的网络（无论是自定义的还是默认网络）都会被清理掉。
* 保留关键数据：这是最重要的特性——所有数据卷（volumes）都会被保留，确保你的数据库文件等重要数据不会丢失。同时，任何标记为 `external: true` 的外部网络和卷也不会被删除。

## 使用示例

### 1. 标准清理

```bash
docker compose down
```

* 这种用法最常用，也最安全。只删除容器和网络，保留所有数据卷，适合日常开发后的清理。

### 2. 彻底重置

```bash
docker compose down -v
```

当你需要将项目环境恢复到最原始状态时，可以加上 `-v` 选项，它会永久删除所有数据。

### 3. 指定服务

```
docker compose down web
```

如果你的项目中有多个服务，只想清理其中一个（比如 `web` 服务），可以直接指定服务名。
