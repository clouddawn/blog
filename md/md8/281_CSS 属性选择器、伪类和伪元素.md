# CSS 属性选择器、伪类和伪元素

## 属性选择器

css属性选择器匹配那些具有特定属性或属性值的元素。

```css
/* 存在 title 属性的 <a> 元素 */
a[title] {
  color: orange;
}

/* 存在 href 属性并且属性值匹配"https://www.baidu.com"的 <a> 元素 */
a[href="https://www.baidu.com"]
{
  color: orangered;
}

/* 存在 href 属性并且属性值包含"example"的 <a> 元素 */
a[href*="http:"]{
  font-size: 20px;
  font-weight: bold;
  color: black;
}
```

## 动态伪类

### 使用举例

* `a:link` 未访问的链接
* `a:visited` 已访问的链接
* `a:hover` 鼠标挪动到链接上
* `a:active` 激活的链接（鼠标在链接上按住还未松开时）

### 使用注意

* `:hover` 必须放在 `:link` 和 `:visited` 后面才能完全生效
* `:active` 必须放在 `:hover` 后面才能完全生效
* 建议编写顺序：`:link  :visited  :hover  :active`
* 除了a元素，`:hover` 和 `:active` 也能用在其他元素上

```css
  a:link {
    color: blue;
  }
  a:visited {
    color: grey;
  }
  a:hover {
    color: red
  }
  a:active {
    color: black;
  }
  h1:hover {
    color: red;
  }
  h1:active {
    color: blue;
    font-size: 15px;
  }
```

## 伪元素

`::first-line` 可以针对首行文本设置属性

`::first-letter` 可以针对首字母设置属性 

```css
.long::first-line {
  background: orangered;
}
.long::first-letter {
  color: blue;
  font-weight: bold;
  font-size: 30px;
}
```

伪元素存在的核心意义，可以用一句话概括：**在不污染 HTML 结构的前提下，弥补 CSS 样式控制力的不足。**

为了让你更透彻地理解，我从四个维度来拆解它存在的必要性：

### 1. 分离“内容”与“表现”（最根本的哲学）

HTML 负责结构（内容层），CSS 负责表现（样式层）。如果为了某个视觉效果（比如给段落首字母放大、在文字前加个图标），我们不得不在 HTML 里额外嵌套 `<span>` 或插入空标签，这就违背了“结构与表现分离”的原则。

- **没有伪元素**：你必须在 HTML 中手动添加装饰性元素，导致 HTML 变得臃肿、语义混乱。
- **有了伪元素**：CSS 自己搞定装饰，HTML 保持纯净的语义（只放真正的数据文本）。

### 2. 访问“不可见”或“不可包裹”的细节

在 CSS 选择器的能力范围内，有些元素细节是无法通过常规选择器定位到的，因为 CSS 没有“选中文本第几个字”或“选中换行后的第一行”的选择器。

- **::first-letter 和 ::first-line**：文本流中的首字母和首行，在 DOM 树中并不存在独立的标签包裹它们。**伪元素硬生生从文本节点中“切”出了一块虚拟区域**，让你能控制排版细节（如首字母下沉、首行缩进或变色），这是纯 CSS 无法单独完成的壮举。

### 3. 实现“纯 CSS”的复杂 UI（减少 HTTP 请求与 JS 依赖）

借助 `::before` 和 `::after` 配合 `content` 属性，伪元素充当了“虚拟子元素”的角色。

- **图标库（如 Font Awesome）**：直接通过伪元素的 `content` 加载 Unicode 字符或 SVG，无需在 HTML 中堆砌 `<i>` 或 `<span>` 标签。

- **经典布局辅助（清除浮动）**：

  ```css
  .clearfix::after {
    content: "";
    display: table;
    clear: both;
  }
  ```

  如果没有伪元素，你必须在 HTML 末尾强行添加一个无意义的 `<div style="clear:both">` 来撑开父容器。伪元素巧妙地避免了这种“冗余标签”。

- **装饰遮罩/角标**：在不增加 DOM 节点的情况下，实现小红点、带角标的对话框等复杂图形。

### 4. 动态性与可维护性

由于伪元素的 `content` 属性支持 `attr()` 函数，它能读取 HTML 属性的值。

```css
/* 当鼠标悬停时，显示链接的提示文字，无需 JS 参与 */
a[data-tooltip]:hover::after {
  content: attr(data-tooltip);
  /* 加上定位和背景，变成自定义气泡提示 */
}
```

这意味着你可以把样式逻辑完全封装在 CSS 文件中。如果将来要修改提示效果，**开发者只需要修改 CSS，完全不用动 HTML 和 JavaScript**，大大降低了维护成本。

------

**总结一下：**
伪元素不是为了“炫技”而存在的，它是 CSS 作为一门“样式语言”走向成熟的标志——**它允许开发者在不破坏文档语义和结构的前提下，将视觉修饰、排版微调和布局辅助，优雅地归入样式表的管辖范围。**

如果非要打个比方：**HTML 是房子的钢筋混凝土（主体结构），伪元素就是墙上的腻子和壁纸（装饰层），你可以随意更换壁纸，但不需要为了贴壁纸而砸墙。**
