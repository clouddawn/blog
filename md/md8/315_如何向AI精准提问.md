# 如何向 AI 精准提问？

向 AI 精准提问是一门艺术，更是一项重要的工程技能。在前端开发中，精准的提问能让你获得更高质量、更可用的代码。



## 提问的核心原则：扮演「产品经理 + 技术主管」的角色

### 1. 提供充足的上下文信息

**❌ 模糊提问：**

```
"写一个表格组件"
```

**✅ 精准提问：**

```
"使用 Vue 3 + Composition API + Element Plus，实现一个用户管理表格组件，要求：
- 支持分页，每页显示10条
- 支持按姓名和邮箱搜索
- 表格列包括：ID、姓名、邮箱、创建时间、操作列
- 操作列有编辑和删除按钮
- 需要显示加载状态
- 使用 TypeScript
请提供完整代码"
```

### 2. 使用结构化描述

**❌ 线性描述：**

```
"我要一个登录表单，有用户名密码，要验证，要提交，要有记住我功能，还要响应式"
```

**✅ 结构化描述：**

```
"开发一个 Vue 3 登录组件，具体要求如下：

## 功能需求
- 用户名/密码输入框
- 记住我复选框
- 提交按钮
- 表单验证

## 技术要求
- 使用 Composition API + `<script setup>`
- 使用 Vuelidate 进行表单验证
- 响应式设计，移动端友好

## 验证规则
- 用户名：必填，邮箱格式
- 密码：必填，最少6位
- 提交时显示加载状态

请提供完整的单文件组件代码"
```



## 前端专属的精准提问技巧

### 3. 指定技术栈细节

**❌ 通用提问：**

```
"实现一个文件上传"
```

**✅ 技术栈明确的提问：**

```
"使用 Vue 3 + Element Plus 的 el-upload 组件，实现一个图片上传功能：
- 支持多选，最多5个文件
- 每个文件不超过2MB
- 只允许 jpg、png 格式
- 上传前进行文件类型和大小校验
- 显示上传进度
- 上传成功后在下方预览缩略图
- 支持删除已上传图片
请提供完整组件代码"
```

### 4. 描述预期的数据流和状态

**❌ 只关注 UI：**

```
"写一个购物车组件"
```

**✅ 包含数据流描述：**

```
"开发一个 Vue 3 购物车组件，数据流要求：

## 状态管理
- 使用 Pinia 管理购物车状态
- 状态包括：商品列表、总价、选中状态

## 组件接口
Props:
  - cartItems: Array<{id, name, price, quantity, image}>
  - loading: boolean

Events:
  - update-quantity(itemId, newQuantity)
  - remove-item(itemId)
  - checkout()

## 功能
- 显示商品图片、名称、单价、数量选择器、小计
- 实时计算总价
- 支持修改数量、删除商品
- 结算按钮

请实现这个组件及对应的 Pinia store"
```

### 5. 提供错误场景和边界条件

**❌ 只考虑正常流程：**

```
"实现数据获取的hook"
```

**✅ 包含异常处理：**

```
"创建一个 Vue 3 Composable `useFetch`，要求：

## 核心功能
- 接收 URL 和配置参数
- 自动管理 loading、data、error 状态

## 边界情况处理
- 请求超时（5秒超时）
- 网络错误处理
- 404 等 HTTP 错误码处理
- 组件卸载时取消未完成请求

## 返回值
{
  data: Ref<T | null>,
  loading: Ref<boolean>,
  error: Ref<Error | null>,
  execute: (url?: string) => Promise<void>
}

请提供完整实现，包含 TypeScript 类型定义"
```



## 高级提问模式

### 6. 分步骤复杂任务

对于复杂功能，拆分成多个步骤：

bash

```
"分三步实现一个实时搜索功能：

## 第一步：基础搜索组件
创建 SearchInput.vue，包含：
- 输入框，支持输入搜索词
- 防抖处理，300ms后触发搜索
- 清除按钮

## 第二步：搜索结果展示
创建 SearchResults.vue，显示：
- 加载状态
- 空状态
- 结果列表
- 错误状态

## 第三步：组合成完整功能
将两个组件组合，实现完整的实时搜索
使用 Vue 3 Provide/Inject 管理搜索状态"
```



### 7. 要求代码质量

```
"实现一个模态框组件，要求：

## 代码质量要求
- 使用 TypeScript，定义完整的接口类型
- 遵循 Vue 3 最佳实践
- 组件要求可复用、可配置
- 包含详细的 JSDoc 注释
- 支持无障碍访问（ARIA 属性）

## 功能配置
- 支持自定义标题、内容、底部按钮
- 支持多种尺寸（sm, md, lg）
- 支持点击遮罩层关闭
- 支持键盘 ESC 关闭

请提供高质量的实现"
```

## 避免的常见错误

1. **❌ 一次问太多** → ✅ 拆分成多个具体问题
2. **❌ 使用模糊术语** → ✅ 使用具体的技术名词
3. **❌ 忽略技术版本** → ✅ 明确 Vue 3、Vue 2 等版本信息
4. **❌ 不提供代码上下文** → ✅ 提供相关代码片段
5. **❌ 不验证结果** → ✅ 总是测试和验证 AI 给出的代码

## 总结

精准提问的本质是**为 AI 提供充足的、结构化的上下文信息**，让 AI 能够准确理解你的：

- **技术环境**（Vue 3、JS/TS、UI 库版本）
- **具体需求**（功能列表、交互细节）
- **质量要求**（代码规范、性能要求）
- **约束条件**（浏览器兼容、移动端适配）

掌握这项技能，AI 将成为你前端开发中最得力的助手，而不是一个只会生成通用代码的工具。
