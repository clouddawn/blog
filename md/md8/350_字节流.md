# 什么是字节流？

字节流（Byte Stream），通俗来讲就是：像水管里的水流一样，持续不断地传输原始二进制数据（0和1组成的序列）。

## 核心本质：流水线 vs 大桶水

* 非流式（大桶水）：必须等所有数据全部加载完，才能拿到完整的 `ArrayBuffer`。比如老式的 `XMLHttpRequest` 下载文件，如果文件有 1GB，浏览器内存瞬间爆炸，因为要把这个文件塞进内存。
* 字节流（流水线）：数据是一块一块（Chunk）到达的。服务器发来 1GB 数据，浏览器收到第 1KB 就能立刻处理，后面的数据还在路上。这些 Chunk 在 JavaScript 中通常以 `Uint8Array` （8位无符号整数数组）的形式存在，每个元素就是一个字节（0~255）。

## Fetch API + 流式下载

假设我们在做一个大文件下载管理器，需要实时显示下载进度，并且边下载边解析（比如解析一个超大 CSV 或预览视频）。

```javascript
// 使用 Fetch 读取字节流
async function downloadAndProcessLargeFile(url) {
  const response = await fetch(url);
  
  // 核心点：response.body 就是一个 ReadableStream（可读字节流）
  const reader = response.body.getReader();
  
  let receivedLength = 0;
  const contentLength = +response.headers.get('Content-Length');

  while (true) {
    // 从流中读取下一块数据（Chunk）
    const { done, value } = await reader.read();
    
    // value 是什么？是 Uint8Array！这就是“字节流”在 JS 中的具体表现
    if (done) break;

    // 处理当前这一小块字节数据
    // 场景1：计算实时进度
    receivedLength += value.length;
    console.log(`下载了 ${receivedLength}/${contentLength} 字节`);
    
    // 场景2：边下边解析（比如这里是视频解码器或文本拼接）
    // 注意：不能直接用 value 当字符串，需要解码或传给 WebAssembly
    // 例如：const textChunk = new TextDecoder().decode(value);
    // 处理这一小段数据...
    
    // 释放内存（重要！）
  }
}
```

**为什么这么写？** 这里 `value` 就是**字节流的分块**。如果不用流，`response.arrayBuffer()` 会等全部下载完才返回，而上述代码在下载完第 1 个 Chunk 时，浏览器就已经拿到原始字节并执行回调了。

## 构建字节流（TransformStream）

前端可以像“管道工”一样操作字节流。比如我们上传图片前，想把图片压缩或添加水印，可以操作字节流：

```javascript
// 把字符串变成字节流发送给服务器
function stringToByteStream(str) {
  // 将字符串编码为 Uint8Array（字节流）
  const encoder = new TextEncoder();
  const uint8Array = encoder.encode(str); // 这里变成了 [72, 101, 108, ...]
  
  // 创建可读字节流
  return new ReadableStream({
    start(controller) {
      controller.enqueue(uint8Array); // 推入字节块
      controller.close();
    }
  });
}

// 使用场景：WebSocket 发送二进制数据，或者 fetch 的 body 传参
fetch('/api/upload', {
  method: 'POST',
  body: stringToByteStream('Hello World'), // 此时 body 就是字节流
  headers: { 'Content-Type': 'application/octet-stream' }
});
```

## 最容易踩的坑

- **编码问题**：字节流是纯二进制，它**不是字符串**。如果你把 `Uint8Array` 直接 `console.log` 看到的是数字数组（如 `[72, 101]`），必须用 `TextDecoder` 解码成文本，或者交给 `Blob` 生成文件。
- **背压（Backpressure）**：如果读取字节流的速度（生产）快于处理速度（消费），浏览器会积压内存。好的流式处理必须使用 `reader.read()` 的异步循环，并在 `finally` 中调用 `reader.releaseLock()` 释放锁，防止内存泄漏。
- **兼容性**：现代浏览器（Chrome 80+）全部支持，但如果要兼容 IE，需要引入 `web-streams-polyfill` 垫片。

------

## 总结

**字节流就是“按时间顺序依次到达的 Uint8Array 数据块”**。在前端，它让我们能够**边收边处理**大文件、**实时预览**视频流、**动态生成**二进制数据（如 Protobuf 协议传输），彻底告别“等所有数据加载完再操作”的低效时代。
