# python 中的命名空间

命名空间是 Python 用来管理变量名与对象之间映射关系的容器。不同的命名空间相互独立，同一个名称可以在不同的命名空间里指向不同的对象，互不干扰。

## 生活中的类比

想象一栋大楼里的多个房间，每个房间里都有一张桌子。房间号就是命名空间的名字，桌子的标签（比如水杯）就是变量名，水杯本身就是对象。

* 在 101 房间，水杯可能是一个玻璃杯。
* 在 202 房间，水杯可能是一个保温杯。

你说水杯，别人必须先知道你在哪个房间里，才能找到对应的那个杯子。

## Python 中的主要命名空间

* 内置命名空间：存放 Python 自带的名字（如 `print`、`len` ），Python 启动时创建。
* 全局命名空间：每个模块（文件）有自己的全局命名空间，模块顶层定义的变量、函数都属于它。
* 局部命名空间：函数或方法内部的名字，每次函数调用时会临时创建。

```python
# 全局命名空间中的 x
x = 10

def foo():
    # 局部命名空间中的 x
    x = 20
    print("局部 x =", x)

foo()          # 输出 局部 x = 20
print("全局 x =", x)  # 输出 全局 x = 10
```

这里的两个 `x` 互不干扰，因为它们属于不同的命名空间。

## 与 `import` 的关系

当你执行 `import math_tools` 时：

- Python 会创建一个名为 `math_tools` 的模块对象，并且这个对象拥有自己独立的全局命名空间。
- 你在当前模块中用 `math_tools.add` 访问 `add`，Python 就会去 `math_tools` 这个命名空间里查找 `add`，而不是在当前模块里找。

> 简单总结：命名空间就是“名字的居所”，让同名变量能够和平共处。
