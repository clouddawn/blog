# docker rmi

`docker rmi` 是 Docker 中用于删除本地镜像的核心命令，它可以帮助你清理不再需要的镜像，释放磁盘空间。

`rmi` 是 remove image 的缩写，用于删除一个或多个本地存储的 Docker 镜像。

## 基本语法

```bash
docker rmi [选项] 镜像名 [镜像名...]
```

其中 `镜像名` 可以是：

- 仓库名 + 标签（如 `nginx:latest`）
- 镜像 ID（如 `f0b7a5b3e4c1`）
- 镜像摘要（digest）

## 常用选项

| 选项          | 说明                                                         | 使用场景                                         |
| ------------- | ------------------------------------------------------------ | ------------------------------------------------ |
| `-f, --force` | 强制删除镜像，即使有容器正在使用它（可能导致正在运行的容器异常） | 当你确信不需要该镜像，且有容器依赖时（谨慎使用） |
| `--no-prune`  | 不删除未加标签的父镜像（即保留中间层）                       | 当你只想删除指定的标签，而保留底层依赖时         |

注意：`-f` 并不能删除正在运行的容器所引用的镜像。如果容器正在运行，即使加上 `-f` 也会失败。你需要先停止并删除容器，或使用 `docker rmi -f` 尝试（仍可能失败）。更安全的方法是先清理容器。

## 示例

### 1. 通过镜像 ID 删除

```bash
docker rmi f0b7a5b3e4c1
```

### 2. 通过仓库名和标签删除

```bash
docker rmi nginx:latest
```

### 3. 同时删除多个镜像

```bash
docker rmi myapp:1.0 myapp:2.0 ubuntu:20.04
```

### 4. 强制删除（谨慎使用）

```bash
docker rmi -f nginx:alpine
```
