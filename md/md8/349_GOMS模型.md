# **GOMS模型**

------

### 1. 核心定义

**GOMS模型** 是由卡内基梅隆大学的学者提出的量化分析方法。它的全称是：

- **G（Goals，目标）**：用户最终想达成什么？（比如“保存这张图片”）
- **O（Operators，操作）**：为了达成目标，用户执行的基本动作有哪些？（比如“点击右键”、“移动鼠标”）
- **M（Methods，方法）**：达成目标的具体操作步骤组合是什么？（比如“方法A：右键→另存为；方法B：Ctrl+S”）
- **S（Selection rules，选择规则）**：在多种方法并存时，用户依据什么条件选用哪种方法？（比如“如果手在键盘上，用Ctrl+S；如果手在鼠标上，用右键”）

GOMS模型就是把用户完成一个任务的所有动作拆成最小单元，然后加起来算总时间，从而客观判断哪个交互流程更快。

------

### 2. 极简版“击键层模型（KLM）”

GOMS模型有一个特别落地的简化版，叫**击键层模型（Keystroke-Level Model, KLM）**。它把人类交互动作拆解成几个固定的“时间片”，你只需要数数就行：

| 操作类型          | 缩写 | 耗时（秒） | 解释                                                 |
| :---------------- | :--- | :--------- | :--------------------------------------------------- |
| **K（击键）**     | K    | 0.20       | 敲一下键盘（打字/回车）                              |
| **P（指向）**     | P    | 1.10       | 鼠标/手指移动到目标位置（**这里套用菲茨定律**）      |
| **H（归位）**     | H    | 0.40       | 手在键盘和鼠标之间来回切换                           |
| **M（心理准备）** | M    | 1.20       | 停下来思考、回忆下一步该干嘛（**这里套用希克定律**） |
| **R（响应）**     | R    | —          | 系统加载/反应时间（视情况而定）                      |

**实操公式**：**总耗时 = K + P + H + M + R**（把所有动作的时间加起来）。

------

### 3. 它和菲茨、希克是什么关系？

- **菲茨定律（Fitts）** 被塞进了 **P（指向）** 的计算里。P 不是固定的1.1秒，如果按钮又小又远，P 会变得很长。
- **希克定律（Hick）** 被塞进了 **M（心理准备）** 的计算里。如果选项太多，用户愣神的时间（M）会变长。

所以，**GOMS模型是“总司令”**，它负责估算总时间；**菲茨和希克是“特种兵”**，负责优化GOMS里的某个具体环节。

------

### 4. 经典对比案例

**场景**：用户在电脑上把一张网页图片保存到桌面。

- **方案A（右键大法）**：手在鼠标上 → **P**（移到图片上）1.1s → **H**（手回键盘？不，不用）0s → **M**（思考一下右键菜单在哪）1.2s → **P**（移到“另存为”）1.1s → **K**（敲回车保存）0.2s → **P**（移到“保存”按钮）1.1s → **K**（回车确认）0.2s。**总计 ≈ 4.9秒**
- **方案B（快捷键大法）**：手在键盘上 → **H**（移到鼠标？不，不用）0s → **M**（回忆快捷键是Ctrl+S）1.2s → **K**（Ctrl）0.2s → **K**（S）0.2s → **M**（思考保存路径弹窗）1.2s → **K**（回车确认）0.2s。**总计 ≈ 3.0秒**

**结论**：通过GOMS算账，方案B（快捷键）快了将近2秒。如果你在评审会上甩出这张测算表，产品经理和开发都会心服口服。

------

### 5. 最重要的误区与禁忌

- **误区一：M（心理准备）永远是最贵的！**
  你看上面的表，M（1.2秒）比K（0.2秒）贵了整整6倍。也就是说，**让用户“愣一下”比让用户“点一下”的代价高得多。**
  *结论*：两个方案对比时，**少一个步骤，不如少一次思考**。方案A需要点5下但全是本能反应（0误判），方案B只需要点3下但每次都要看文字辨认（高认知）。根据GOMS，**方案A（点5下）往往比方案B（点3下但想3次）更快**。这打破了“点击次数越少越好”的肤浅认知。
- **禁忌：GOMS只适合“专家用户”和“高频任务”**
  如果用户是第一次用这个功能（新手），或者这个功能一年只用一次（低频），那么 **M（心理准备）的时间会无限放大**，甚至直接放弃操作。此时，就算GOMS算出方案A快，但用户找不到入口，方案A就是废的。**对于新手和低频任务，“可见性”（示能性）比“效率”（GOMS）重要一万倍。**
- **陷阱：算总时长，但别忽略“打断感”**
  GOMS假设用户是一个连贯执行任务的机器。但如果方案里突然冒出一个弹窗打断了用户的“心流”（比如强制看5秒广告），那这个打断带来的负向体验，是GOMS里那几秒时间完全无法衡量的。此时要结合**心流理论**（Csikszentmihalyi）来弥补GOMS的盲区。

------

### 6. 实战自检清单

当你拿着A、B两个流程方案，别再只数“几步”了，拿张草稿纸：

1. **拆解原子动作**：把两个方案都拆成 K、P、H、M。（尤其注意 **H**——手在键盘和鼠标之间来回挪动，这个时间经常被忽略，但极其昂贵）。
2. **标记“思考点”**：圈出流程里所有需要用户**停下来读文字、回忆、判断**的地方（即M）。哪个方案的M少，哪个方案大概率胜出。
3. **区分“新手”还是“老手”**：如果这个功能针对专业软件（如PS、Excel）的老手，无脑选GOMS最快的（快捷键）；如果针对广场舞大妈，放弃GOMS，选**文字最大、解释最清楚**的那个（哪怕多两步）。

------

### 最后送你一句打通三者的心法

- **希克定律**：解决“眼睛看不过来”的问题（**少选**）。
- **菲茨定律**：解决“手指够不过去”的问题（**近大**）。
- **GOMS模型**：解决“脑子+手整体算账”的问题（**少想少动**）。

**终极结论**：当GOMS算出来A比B快，但A让用户觉得“心里没底”（焦虑）时——**选B**。因为GOMS算的是“物理时间”，而用户感知的“心理时间”比物理时间重要10倍。好的交互是“虽慢但安心”，不是“虽快但慌张”。
