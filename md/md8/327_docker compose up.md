# docker compose up

`docker compose up` 是 Docker Compose  中最核心的一个命令。它不是用来启动一个新服务的，而是将服务创建、启动到运行的全流程整合在了一起，负责将你的整个应用环境唤醒。

它的工作流程如下：

1. 检查并构建镜像：如果服务需要构建（在 `compose.yaml` 中定义了 `build`），它会先执行一次 `build` 操作。
2. 拉取缺失镜像：如果服务依赖远程镜像且本地不存在，它会自动执行 `pull` 操作。
3. 创建并启动容器：为所有服务创建新容器并启动。
4. 自动处理依赖：会同时启动所有依赖的服务（使用 `depends_on` 定义），确保你的服务间不会因启动顺序出错。

| 命令模式         | 运行行为               | 日志输出                                     | 停止方式                                      | 使用场景                                   |
| ---------------- | ---------------------- | -------------------------------------------- | --------------------------------------------- | ------------------------------------------ |
| 默认模式（前台） | `docker compose up`    | 终端会附加到所有容器，实时显示聚合的日志流。 | 按 `Ctrl + C` 会停止所有运行中的容器。        | 本地开发、调试代码、观察服务启动详情       |
| 分离模式（后台） | `docker compose up -d` | 不显示日志，容器在后台持续运行。             | 需要使用 `docker compose down` 命令手动停止。 | 生产环境、长期运行的服务、需要解放当前终端 |

```bash
# 指定单个服务
docker compose up web
```
