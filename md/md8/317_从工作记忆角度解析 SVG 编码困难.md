# 从工作记忆角度解析 SVG 编码困难

我在用AI编程前端代码时，AI在实现一个图表时使用了svg，而我不懂svg编码，这导致我修改这段代码很困难。

### 1. 符号系统不熟悉导致认知负荷倍增

当看到自己熟悉的HTML/CSS时：

```html
<div class="box" style="width:100px; height:100px; background:red;"></div>
```

* `div`、`class`、`style` 都是已组块化的符号，我可以瞬间理解，大脑几乎不消耗工作记忆就能处理。

当我看到陌生的 SVG 时：

```html
<path d="M10 10 C 20 20, 40 20, 50 10" stroke="black" fill="transparent"/>
```

* `path`、`d`、`c` 等符号对我来说都是新的、未组块化的信息单元。
* 每个符号都需要占据单独的工作记忆槽位。
* 大脑需要额外资源来解码这些符号的基本含义。

### 2. 心智模型缺失导致工作记忆无法建立有效连接

在熟悉技术中：

```css
.box {
  width: 100px;
  height: 100px;
  border-radius: 50%;
}
```

我的心智模型是：

```
圆角半径 = 50% → 高度/宽度的一半 → 变成圆形
```

这个推理链条在长期记忆中已形成自动模式，几乎不占用工作记忆。



在陌生 SVG 中：

```html
<path d="M10 80 Q 95 10 180 80" stroke="black" fill="transparent"/>
```

我的心智模型是空白的，需要现时推导：

```text
d → 路径数据 → M是移动 → Q是二次贝塞尔曲线 → 贝塞尔是什么？ → 控制点在哪？
```

每一步都需要现时计算，每一步都占据工作记忆，且前一步的结论可能在思考后一步是被挤出。

### 3. 双重任务导致工作记忆过载

当我尝试修改SVG代码时，工作记忆需要同时处理：

**任务1：解码陌生语法**

```text
工作记忆槽位1: M = moveTo (起点)
工作记忆槽位2: Q = 二次贝塞尔曲线  
工作记忆槽位3: 参数含义？(控制点x, 控制点y, 终点x, 终点y)
工作记忆槽位4: 如何修改才能得到想要的效果？
```

**任务2：维持设计目标**

```text
工作记忆槽位5: 我想要一个更平滑的曲线
工作记忆槽位6: 颜色需要改成蓝色
工作记忆槽位7: 这个图标需要响应式
```

工作记忆通常只有 **4±1** 个槽位，现在需要同时处理7个相互关联但未组块化的任务，必然导致：

* 顾此失彼：专注于理解语法时忘记了设计目标
* 频繁切换：在语法解码和目标实现间来回切换，每次切换都有信息丢失
* 认知疲劳：大脑不断重新加载被挤出的信息，消耗大量能量

假设想修改 SVG 的大小：

```html
<!-- 你想把图标放大，但改了width/height后形状变形了 -->
<svg width="100" height="100" viewBox="0 0 24 24">
  <path d="M12 2L15.09 8.26..."/>
</svg>
```

工作记忆需要同时保持：

* 问题现象：图标变形了
* 可能的假设：`viewBox` 可能与尺寸相关
* 相关知识检索：从长期记忆中搜索 `viewBox` 是什么？因为没学过，所以是空白。
* 验证方案：尝试修改 `viewBox` 并观察效果
* 结果跟踪：记住每次修改后的变化，与之前对比

但实际中：

* 修改了 width ==> 发现变形  ==>  假设是 `viewBox` 问题
* 查询 `viewBox` ==>  理解概念  ==>  尝试修改
* 在修改过程中，之前为何要修改 width 的上下文被挤出了
* 忘记了最初的意图，陷入盲目试错循环

### 4. 缺乏模式识别导致无法形成认知捷径

熟悉css时，你一眼能看出：

```css
.container {
  display: flex;
  justify-content: center;
  align-items: center;
}
```

这是垂直水平居中的模式，不需要逐个解析属性，它们作为一个整体模式被识别。

面对svg时，则没有这种模式识别：

```html
<g transform="translate(50, 50) rotate(45) scale(1.5)">
  <rect x="10" y="10" width="50" height="30"/>
</g>
```

每个属性都需要单独解析、理解、记住，无法形成整体认知模式。
