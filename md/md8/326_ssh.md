# 什么是 SSH ？

SSH 是一种加密的网络协议，用于在不安全的网络中安全地访问远程计算机。它主要用来：

* 远程登录：获得一个远程主机的命令行界面
* 远程执行命令：不登录完整 shell ，直接在远程机器上运行一条命令
* 安全的文件传输：`scp`、`sftp` 等工具都基于 SSH 加密传输
* 端口转发/隧道：把本地端口映射到远程服务器，或反之（常用于穿透内网、加密数据库连接等）

## 核心原理

SSH 采用 客户端-服务器 模型：

*  服务器端运行 `sshd` 服务，监听某个端口
* 客户端用 `ssh` 命令发起连接
* 连接时通过非对称加密交换密钥，建立加密通道，后续通信全部加密

认证方式常见有两种：

* 密码认证：输入用户密码
* 密钥认证（公钥/私钥对）：把公钥放到服务器，客户端持有私钥，无需密码更安全

## 基本用法

```bash
ssh [选项] [用户名@]主机 [命令]
```

* **远程登录**

  ```bash
  ssh -p 1435 root@10.34.0.171
  ```

  登录后你会进入服务器的 shell ，可以随意操作。

* **不登录，直接执行一条命令**

  ```bash
  ssh root@10.34.0.171 "ls -l /home"
  ```

  执行完命令后自动退出，结果返回本地终端。

* 使用密钥登录

  ```bash
  ssh -i ~/.ssh/mykey user@hose
  ```

  配合密钥，可以实现免密码登录。

## 常用选项

| 选项              | 说明                           |
| ----------------- | ------------------------------ |
| `-p 端口`         | 指定远程 SSH 端口（小写 p）    |
| `-i 私钥文件`     | 指定身份验证的私钥路径         |
| `-v / -vv / -vvv` | 详细调试信息，用于排查连接问题 |

## SSH 配置文件（简化连接）

每次都输入端口、IP、用户名、密钥很麻烦，可以编辑 `~/.ssh/config` 文件，定义别名：

```text
Host my_server
	HostName 10.34.0.171
	Port 1435
	User root
	IdentifyFile ~/.ssh/id_ed25519
```

然后直接 `ssh my_server` 即可，甚至 `scp my_server:/path/...` 也通用。

## SSH 与 scp、sftp 的关系

* **scp：**基于 SSH 协议的文件复制命令，语法类似 `cp`
* **sftp：**交互式文件传输（像 FTP 一样），底层也是 SSH
* **ssh：** 最核心的客户端，提供远程 shell、命令执行、端口转发
