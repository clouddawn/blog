  * **cp 1.txt 2.txt**

## 命令提示符 vs PowerShell

命令提示符和 PowerShell 是 Windows 系统中两个核心的命令行工具，它们都能让你通过文本命令来控制电脑，但设计思想和能力有很大不同。

可以把它们理解为：

- **命令提示符**：一个传承自 DOS 时代的经典工具，简单、直接，适合执行基础任务。
- **PowerShell**：一个现代化的强大自动化平台，能深入管理系统的方方面面。

### 命令提示符 (Cmd)

- **本质**：一个**文本界面**的命令解释器。它模拟了旧式DOS（磁盘操作系统）的操作环境。
- **核心语言**：批处理语言。你需要把命令一行行输入，或写成 `.bat` 脚本文件。
- **擅长领域**：
  - 执行基础的网络命令（如 `ping`, `ipconfig`, `tracert`）。
  - 运行系统文件检查工具（如 `sfc /scannow`）。
  - 调用一些传统的管理工具（如 `diskpart`）。
  - 执行一些简单的文件操作（如 `copy`, `dir`, `del`）。
- **优点**：
  - **轻量快速**：启动和运行都很快。
- **缺点**：
  - **能力有限**：无法直接操作现代的许多系统核心（如注册表、WMI）。
  - **输出是文本**：它把命令输出都当作纯文本，很难进一步用其他命令处理。比如，你用 `dir` 列出一个文件列表，想从中找出所有超过1MB的文件，用命令提示符本身几乎做不到。

### PowerShell

- **本质**：一个**任务自动化**和**配置管理**框架。它远不止是一个命令行工具，更是一个完整的脚本环境。
- **核心语言**：基于.NET Framework的脚本语言。它的命令遵循“动词-名词”的命名规则，比如 `Get-Process`, `Stop-Service`。
- **擅长领域**：
  - **系统管理**：可以管理系统服务、注册表、事件日志、进程等几乎所有Windows组件。
  - **管理微软云服务**：是管理Exchange Online、Microsoft 365、Azure等云平台的官方工具。
  - **复杂数据处理**：可以像处理数据库一样处理文本、CSV、JSON、XML等数据。
  - **编写强大脚本**：能实现复杂的逻辑，替代传统的图形界面操作。
- **优点**：
  - **面向对象**：这是最核心的区别。PowerShell传递的不是文本，而是“对象”。对象就像一张详细的资料卡，包含了某个东西（比如一个进程）的所有属性和方法。这让数据处理变得极其直观和强大。例如，用 `Get-Process | Where-Object { $_.CPU -gt 100 }` 就能轻松找出CPU占用超过100的进程。
  - **管道无比强大**：因为传递的是对象，所以 `|` 管道符前后的命令可以无缝衔接，不会丢失信息。
  - **统一管理**：无论你要操作文件、注册表、证书还是IIS网站，用的都是一套统一、逻辑清晰的命令。
- **缺点**：
  - **学习曲线稍陡**：命令多，且“面向对象”的概念对习惯传统命令行的人需要适应。
  - **启动稍慢**：因为它需要加载.NET运行时环境，所以首次启动会比命令提示符慢一点。

### 核心区别对比表

| 特性         | 命令提示符 (Cmd)                         | PowerShell                                            |
| :----------- | :--------------------------------------- | :---------------------------------------------------- |
| **核心概念** | 面向**文本**                             | 面向**对象**                                          |
| **命令风格** | 传统、简短的命令 (如 `ipconfig`, `ping`) | 详细、自解释的“动词-名词”结构 (如 `Get-NetIPAddress`) |
| **管道功能** | 传递文本，能力很弱                       | 传递对象，能力极强，是核心功能                        |
| **脚本能力** | `.bat` 批处理，简单流程控制              | `.ps1` 脚本，完整的编程语言能力                       |
| **管理范围** | 基础文件、网络和系统工具                 | 几乎所有Windows组件、应用和云服务                     |

### 实际使用建议：我该用哪个？

1. **当你需要执行一个明确、简单的命令时，用命令提示符。**
   - 例子：`ping google.com` 检查网络通不通。
   - 例子：`ipconfig /all` 查看自己的IP地址和MAC地址。
2. **当你开始尝试任何自动化、管理或复杂任务时，用PowerShell。**
   - **管理计算机**：想批量重命名100个文件、查看所有运行中进程的详细内存占用、禁用某个不喜欢的服务。
   - **处理数据**：从日志文件中提取所有包含“Error”的行，然后统计每个错误出现的次数。
   - **办公自动化**：用脚本批量生成Excel报表，或向Word文档里填入数据。
   - **管理云资源**：你要在Azure或Microsoft 365上做一些操作。

### 如何打开它们？

- **命令提示符**：按 `Win + R`，输入 `cmd` 然后回车。或者在开始菜单搜索“cmd”或“命令提示符”。
- **PowerShell**：在开始菜单搜索 `PowerShell`。
- **重要提示（管理员权限）**：很多系统级别的操作（比如修改网络配置、安装软件）需要管理员权限。在搜索结果中，右键点击程序图标，选择 **“以管理员身份运行”**。这点对两个工具都一样。

### 总结

- **命令提示符**是一把**小巧耐用的瑞士军刀**，处理日常小任务很方便。
- **PowerShell**则是一个**现代化的精密机床**，威力巨大，能完成各种复杂的自动化工作。

对于Windows用户来说，了解命令提示符有助于处理一些简单任务和兼容老教程，但掌握PowerShell才是通往系统管理高手之路。值得一提的是，在Windows 11 22H2及之后的版本中，命令提示符的默认终端已被替换为**Windows终端**。这是一个现代化、支持多标签页的工具，你可以同时在标签页里运行命令提示符、PowerShell或WSL Linux子系统。

### 二、PowerShell 常用命令（Cmdlet）

PowerShell 的命令遵循 **动词-名词** 命名规范，例如 `Get-Process`、`Stop-Service`。
绝大多数命令都有友好的**别名**（如 `dir`、`ls` 是 `Get-ChildItem` 的别名），但推荐直接使用原生命令以获得完整功能。

### 1. 文件和目录操作（核心：`Item` 系列）

| 原生命令        | 常用别名      | 说明             | 示例                                             |
| :-------------- | :------------ | :--------------- | :----------------------------------------------- |
| `Get-ChildItem` | `dir`, `ls`   | 列出目录内容     | `Get-ChildItem -Recurse`                         |
| `Set-Location`  | `cd`, `chdir` | 切换目录         | `Set-Location C:\`                               |
| `New-Item`      | `ni`          | 创建新文件或目录 | `New-Item -Path . -Name test.txt -ItemType File` |
| `Copy-Item`     | `copy`, `cp`  | 复制文件/目录    | `Copy-Item a.txt D:\ -Force`                     |
| `Move-Item`     | `move`, `mv`  | 移动/重命名      | `Move-Item a.txt D:\`                            |
| `Remove-Item`   | `del`, `rm`   | 删除文件/目录    | `Remove-Item *.log -Recurse`                     |
| `Get-Content`   | `cat`, `type` | 查看文件内容     | `Get-Content log.txt -Tail 10`（最后10行）       |
| `Set-Content`   | `sc`          | 写入文件内容     | `"Hello" | Set-Content file.txt`                 |

### 2. 进程与服务管理

| 命令                             | 说明             | 示例                                        |
| :------------------------------- | :--------------- | :------------------------------------------ |
| `Get-Process`                    | 获取进程列表     | `Get-Process | Sort-Object CPU -Descending` |
| `Stop-Process`                   | 终止进程         | `Stop-Process -Name notepad -Force`         |
| `Get-Service`                    | 查看系统服务状态 | `Get-Service W3SVC`                         |
| `Start-Service` / `Stop-Service` | 启动/停止服务    | `Start-Service Spooler`                     |
| `Restart-Computer`               | 重启计算机       | 需管理员                                    |

### 3. 对象筛选与处理（PowerShell 精华）

| 命令（Cmdlet）   | 说明                         | 示例                                             |
| :--------------- | :--------------------------- | :----------------------------------------------- |
| `Where-Object`   | 筛选对象（相当于SQL的WHERE） | `Get-Process | Where-Object {$_.CPU -gt 100}`    |
| `Select-Object`  | 选择对象的特定属性           | `Get-Service | Select-Object Name, Status`       |
| `Sort-Object`    | 排序                         | `Get-ChildItem | Sort-Object Length -Descending` |
| `ForEach-Object` | 对每个对象执行操作           | `1..5 | ForEach-Object { $_ * 2 }`               |
| `Export-Csv`     | 将数据导出为CSV              | `Get-Process | Export-Csv proc.csv`              |

### 4. 文本与文件内容处理

| 命令                   | 说明                             | 示例                                         |
| :--------------------- | :------------------------------- | :------------------------------------------- |
| `Select-String`        | 在文件或字符串中搜索（类似grep） | `Select-String -Path *.log -Pattern "error"` |
| `Get-Content` + `-Raw` | 读取整个文件                     | `$text = Get-Content file.txt -Raw`          |
| `Out-File`             | 输出到文件                       | `Get-Service | Out-File services.txt`        |

### 5. 系统信息与网络

| 命令                                | 说明                                                      |                                                              |
| :---------------------------------- | :-------------------------------------------------------- | ------------------------------------------------------------ |
| `Get-WmiObject` / `Get-CimInstance` | 查询系统信息（如 `Get-CimInstance Win32_ComputerSystem`） |                                                              |
| `Test-Connection`                   | 类似 ping，但返回对象                                     | `Test-Connection 8.8.8.8 -Count 4`                           |
| `Get-NetIPAddress`                  | 查看IP地址（需管理员）                                    |                                                              |
| `Invoke-WebRequest`                 | 发送HTTP请求（下载网页/文件）                             | `Invoke-WebRequest -Uri https://example.com -OutFile page.html` |

### 6. 帮助与发现命令

| 命令          | 说明                     |                                  |
| :------------ | :----------------------- | -------------------------------- |
| `Get-Command` | 列出所有可用命令         | `Get-Command *Process*`          |
| `Get-Help`    | 查看命令帮助             | `Get-Help Get-Process -Examples` |
| `Get-Member`  | 查看对象有哪些属性和方法 | `Get-Process | Get-Member`       |

### 三、实用场景对比：同一任务的不同做法

| 任务                         | 命令提示符                             | PowerShell（原生风格）                                       |
| :--------------------------- | :------------------------------------- | :----------------------------------------------------------- |
| 列出当前目录所有 `.txt` 文件 | `dir *.txt`                            | `Get-ChildItem -Filter *.txt`                                |
| 查找包含“error”的日志行      | `find "error" log.txt`                 | `Select-String -Path log.txt -Pattern "error"`               |
| 杀死所有记事本进程           | `taskkill /im notepad.exe /f`          | `Get-Process notepad | Stop-Process -Force`                  |
| 显示CPU占用前5的进程         | （需要借助第三方工具或复杂批处理）     | `Get-Process | Sort-Object CPU -Descending | Select-Object -First 5` |
| 列出所有停止的服务           | `sc query state= inactive`（输出混乱） | `Get-Service | Where-Object {$_.Status -eq "Stopped"}`       |

### 四、快速上手指南

- **普通用户/新手**：先掌握 Cmd 的 `dir`, `cd`, `ping`, `ipconfig` 就足以应付很多网络诊断和基础文件操作。
- **系统管理员/开发者**：直接投入 PowerShell。记住三个万能命令：
  - `Get-Command`：搜命令
  - `Get-Help`：看帮助（加 `-Examples` 看例子）
  - `Get-Member`：查对象有什么“零件”（属性/方法）

另外，**Windows 终端**（Windows Terminal）可以同时承载 Cmd、PowerShell 和 WSL，支持多标签页、自定义主题，强烈推荐安装使用。



## Git Bash

Git Bash 是为 Windows 用户准备的一个“宝箱”，它既包含了强大的版本控制工具 Git，又提供了一个轻量级的 Linux 模拟环境。它尤其适合那些想在不离开 Windows 系统的前提下，获得 Linux 般流畅开发体验的程序员。

它主要有以下两大核心能力：

- **Git版本控制**：让你能无缝使用 `git clone`、`git commit` 等全部Git命令。
- **Unix 命令集**：原生支持 `ls`、`pwd`、`grep`、`awk` 等常用命令，让Linux习惯在Windows上延续。

#### 文件与目录操作

| `grep`                | 强大的文本搜索，是日志排查的利器 | `grep "error" app.log`             |
| `find`                | 按文件名等条件搜索文件           | `find . -name "*.txt"`             |
| `curl` / `wget`       | 发送网络请求或下载文件           | `curl https://api.example.com`     |
| `tar` / `zip`/`unzip` | 打包/压缩及解压文件              | `tar -czf archive.tar.gz ./folder` |
| `ps` / `kill`         | 查看/终止系统进程                | `ps aux` 或 `kill 1234`            |

#### Git 核心命令

| 命令                       | 说明                                |
| :------------------------- | :---------------------------------- |
| `git init`                 | 在当前目录初始化一个新的 Git 仓库   |
| `git clone <仓库地址>`     | 将远程仓库完整复制到本地            |
| `git add <文件名>`         | 将文件的更改添加到暂存区            |
| `git commit -m "提交信息"` | 将暂存区的更改提交到本地仓库        |
| `git push` / `git pull`    | 将本地提交推送到远程/从远程拉取更新 |
| `git status` / `git diff`  | 查看文件修改状态/查看具体内容差异   |

### Git Bash vs. CMD vs. PowerShell

| 对比维度       | Git Bash                                               | CMD（命令提示符）                          | PowerShell                                            |
| :------------- | :----------------------------------------------------- | :----------------------------------------- | :---------------------------------------------------- |
| **出身与背景** | 为 Windows 提供的 **Unix/Linux 风格**模拟环境          | Windows 原生，承继自 DOS 的传统命令行      | Windows 原生，基于 .NET 的新一代命令行                |
| **命令语言**   | 支持 `ls`, `pwd` 等 **Unix 命令**，语法与 Linux 一致   | 使用 `dir`, `cd` 等 **DOS 命令**，语法简练 | 支持 `Get-ChildItem` 等 **Cmdlet**，语法为“动词-名词” |
| **核心哲学**   | **文本流**。一切皆文本，命令间通过纯文本管道传递信息。 | **文本流**。核心功能弱，管道能力有限。     | **面向对象**。命令间传递的是 .NET 对象，功能强大      |
| **主要用途**   | **软件开发**、Git操作、需要跨平台兼容性的任务          | **Windows 管理**，执行一些基础、简单的命令 | **Windows 深度管理**、复杂自动化运维脚本              |
